# Фабрика гипотез

Решение для хакатона от Норникеля.

## Проблематика

В промышленных лабораториях и НИИ отсутствует системный механизм генерации гипотез. Это приводит к субъективности отбора идей, слабой связи с бизнес-эффектом, неэффективному использованию исторических данных и замедлению старта проектов.

**Ключевые проблемы:**
- Субъективность и зависимость от экспертов.
- Низкая связность гипотез с бизнес-целями.
- Неиспользование накопленных отчётов и экспериментов.
- Отсутствие единого формата обоснования гипотез.
- Затягивание времени на согласование исследовательских направлений.

## Описание решения

Интеллектуальная система для генерации и приоритизации технологических гипотез на основе загруженных данных. Целевые пользователи — исследователи и технологи обогатительных фабрик.

**Что делает система:**
- Принимает на вход: целевую задачу (например, «снизить себестоимость шихты»), ограничения (бюджет, оборудование) и базу знаний (научные публикации, отчёты, регламенты).
- Формирует на выходе: список проверяемых гипотез с обоснованием, ссылками на источники, оценкой рисков и ожидаемой ценности.
- Опционально: дорожную карту проверки с последовательностью экспериментов и критериями успеха.

**Ключевые требования, которые мы выполняем:**
- **Полезность для исследователей:** гипотезы конкретны и проверяемы в лабораторных условиях.
- **Прозрачность:** каждая рекомендация сопровождается ссылками на источники и обоснованием.
- **Гибкость:** работа с неполными и разнородными данными (PDF, Excel, DOCX, PNG).
- **Масштабируемость:** возможность подключения новых предметных областей без перестройки ядра.
- **Интеграция:** импорт из распространённых форматов и экспорт в Jira/YouTrack, CSV, PDF.

## Архитектура

Система построена на основе RAG (Retrieval-Augmented Generation) и состоит из следующих компонентов:

- **Backend**: обработка данных (парсинг PDF, Excel, DOCX), векторизация текстов, работа с LLM, извлечение сущностей и связей.
- **Frontend**: веб-интерфейс для загрузки файлов, ввода запросов и получения результатов с визуализацией связей.

Подробное описание работы приведено в `src/README.md`.

## Структура репозитория

- `01_Data/` — исходные данные, предоставленные организаторами.
- `02_Analysis/` — промежуточные аналитические выводы, генерируемые системой.
- `03_Results/` — финальные гипотезы и отчеты, генерируемые системой.
- `src/` — исходный код системы.

## Запуск проекта

1. Клонировать репозиторий.
2. Установить зависимости: `pip install -r src/requirements.txt`.
3. Запустить приложение: инструкция в `src/README.md`.
